# Roadmap status at Staqtapp 1.4.0

| Roadmap phase | Status in 1.4.0 |
|---|---|
| Phase 0 — freeze and inventory | Source audit complete; repository history and additional advanced fixtures remain external evidence gaps. |
| Phase 1 — compatibility contract | Stable core signatures and regression tests established. Advanced contracts remain unrecovered. |
| Phase 2 — mapped read foundation | Production mapped snapshot, lazy payload spans, compact indexes, and `.sqti` sidecar implemented for the selected hierarchy. |
| Phase 3 — transaction writer | Atomic replace, backups, conflict detection, and recovery are stable. Streaming range-copy writer remains incomplete. |
| Phase 4 — core variable parity | Stable supported core CRUD and codecs. |
| Phase 5 — hierarchy and embedded files | Existing hierarchy selection supported; complete CRUD not yet implemented. |
| Phase 6 — locks, history, contexts, concurrency | Locks, history, explicit contexts, spawn dispatch, and per-VFS writer locks are present. Broader platform CI remains. |
| Phase 7 — migration and verification | Stable core verification, migration, and backup recovery are present. |
| Phases 8–11 — advanced features | Explicitly unsupported pending contract and security recovery. |
| Phase 12 — consolidation | Core package is production-shaped; full roadmap consolidation is not complete. |

Version 1.4.0 remains a Staqtapp 1.x minor release because the public product contract and SQTPP compatibility are preserved.
